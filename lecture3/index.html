§ 01

왜 Dynamic이 필요한가?

지금까지 분석한 Strategic-Form 게임은 타이밍(timing)이 없다. 모든 플레이어가 동시에 결정하거나, 상대방의 행동을 모르고 결정한다고 가정한다. 하지만 현실의 많은 상황에서는 플레이어들이 순차적으로(sequentially) 움직인다.

Cournot → Stackelberg: 순서가 결과를 바꾼다

비교 예시 — a=1, c=0
Cournot (동시)

두 기업이 동시에 생산량 결정

NE: q₁* = q₂* = 1/3 보수: u₁ = u₂ = 1/9
Stackelberg (순차)

Firm 1이 먼저, Firm 2가 관찰 후 결정

SPNE: q₁* = 1/2, q₂* = 1/4 보수: u₁ = 1/8 > 1/9
💡 First-mover Advantage

순서가 달라지면 균형도 달라진다. Stackelberg에서 Firm 1은 선점 이득(first-mover advantage)을 얻는다. 이를 분석하려면 Extensive Form이 필요하다.

Matching Pennies: 순서가 달라지면?

LN2의 동시 게임 Matching Pennies에서는 순수전략 NE가 없었다. 그런데 Anne이 Bob 다음에 움직인다면? Backward Induction으로 풀면:

  • Bob이 H를 선택하면 → Anne은 H (payoff +1)
  • Bob이 T를 선택하면 → Anne은 T (payoff +1)
  • Bob은 어떻게 해도 −1 → 무차별 → (H,H) 또는 (T,T)
🔑 Second-mover Advantage

Anne이 나중에 움직이면 항상 이긴다 — second-mover advantage! 동시 게임과는 완전히 다른 결과가 나온다.

§ 02

Tree와 Extensive Form 게임 정의

Definition — Tree (트리)

트리는 노드(nodes)들이 방향 있는 호(directed arcs)로 연결된 집합으로, 다음 조건을 만족한다:

  1. 초기 노드(initial node)가 하나 존재한다
  2. 초기 노드 이외의 각 노드에 들어오는 호(arc)가 정확히 하나이다
  3. 각 노드는 유일한 경로(unique path)를 통해 도달 가능하다
Definition — Extensive Form Game

다음 요소들로 구성된다:

  • Players — 플레이어 집합
  • Tree — 게임의 구조를 나타내는 트리
  • Player allocation — 각 비단말 노드(non-terminal node)에 플레이어 배정
  • Information partition — 정보 집합의 분할
  • Payoffs — 각 단말 노드(terminal node)에서의 플레이어별 보수

노드의 종류

종류설명
Initial node게임이 시작되는 유일한 루트 노드 (들어오는 호 없음)
Non-terminal node의사결정이 이루어지는 노드; 특정 플레이어에게 배정됨
Terminal node게임이 끝나는 노드; 보수 벡터가 정의됨 (나가는 호 없음)

Extensive Form 게임 트리 예시

파란 노드: initial node / 금색 노드: P1의 decision node / 초록 노드: P2의 decision node

§ 03

Information Set / Partition

Definition — Information Set (정보 집합)

정보 집합은 다음 조건을 만족하는 노드들의 집합이다:

  • 집합 내 모든 노드에서 동일한 플레이어가 행동한다
  • 집합 내 모든 노드에서 동일한 행동 집합이 가능하다

플레이어가 자신이 어느 노드에 있는지 모를 수 있다 → 두 노드 이상이 점선으로 묶임 = non-singleton information set

Definition — Information Partition (정보 분할)

트리의 모든 비단말 노드를 정보 집합으로 분할(partition)하는 것. 각 노드는 정확히 하나의 정보 집합에 속한다.

순차 게임 (Singleton IS)

Anne이 Bob의 행동을 보고 결정한다. 각 노드가 별도의 정보 집합 — 완전 정보(perfect information).

Anne의 정보집합: {ha(1)}, {ha(2)} (둘 다 singleton)

동시 게임 (Non-singleton IS)

Anne이 Bob의 행동을 모르고 결정. 두 노드가 점선으로 묶임 — 불완전 정보(imperfect information).

Anne의 정보집합: {ha(1), ha(2)} (non-singleton)

공식 표기

h(x) : 노드 x가 속한 정보 집합 A(x) : 노드 x에서 가능한 행동 집합 x' ∈ h(x) : x와 같은 정보 집합 → 같은 플레이어, 같은 행동 집합 즉, A(x') = A(x) A(h') ≡ A(x') = A(x) where h' ≡ h(x') = h(x)
§ 04

전략의 정의: Action vs Strategy

⚠️ 시험 필수 구별!

Extensive Form에서 action(행동)과 strategy(전략)는 다른 개념이다!

개념정의예시
Action 특정 정보집합에서 선택하는 단일 행동 ha(1)에서 "H를 선택"
Strategy 자신이 움직이는 모든 정보집합에 대한 완전한 행동 계획 (contingent plan) {ha(1)→H, ha(2)→T} = "HT"
Definition — Pure Strategy (순수전략)

플레이어 i의 순수전략은 자신의 모든 정보집합 hᵢ ∈ Hᵢ에 행동을 배정하는 함수:

sᵢ : Hᵢ → Aᵢ, with sᵢ(hᵢ) ∈ A(hᵢ) for all hᵢ ∈ Hᵢ 순수전략 공간: Sᵢ = ∏_{hᵢ ∈ Hᵢ} A(hᵢ) (각 IS별 행동집합의 직적)

혼합전략(Mixed strategy) = 순수전략들에 대한 확률 분포

Example — 순차 Matching Pennies에서의 전략

Bob이 먼저 H/T 선택. Anne이 Bob의 행동을 보고 결정.

Anne의 정보집합: ha(1) (Bob이 H 선택 후) 와 ha(2) (Bob이 T 선택 후)

Bob의 순수전략: {H, T} ← 정보집합 1개, 행동 2개 Anne의 순수전략: sA : {ha(1), ha(2)} → {H, T} SA = {HH, HT, TH, TT} ← 4가지 전략 예) "HT" = Bob이 H를 선택하면 H, T를 선택하면 T
💡 핵심

전략은 실제로 도달하지 않을 정보집합에 대해서도 행동을 명시해야 한다. "어떤 상황에서도 내가 무엇을 할지"의 완전한 계획이기 때문이다.

§ 05

Strategic Form vs Extensive Form

모든 Extensive Form 게임은 전략들을 나열하면 Strategic Form(정규형 행렬)으로 변환할 수 있다.

순차 Matching Pennies: Extensive → Strategic

Bob: {H, T} / Anne: {HH, HT, TH, TT}

Bob ↓ / Anne →HHHTTHTT
H−1, +1+1, −1−1, +1+1, −1
T+1, −1+1, −1−1, +1−1, +1
Theorem — 변환의 방향성
  • Extensive Form → Normal Form: 항상 유일하게 변환 가능 (전략을 적절히 명명할 경우)
  • Normal Form → Extensive Form: 역은 성립하지 않음! 하나의 정규형에 여러 Extensive Form이 대응될 수 있음
💡 정규형의 두 가지 해석
  • 게임이 진정으로 동시에 플레이된다
  • 게임이 순차적이지만 동적 구조를 추상화하여 표현한다

정규형으로 변환하면 타이밍 정보가 손실되므로, 동적 측면이 중요한 경우 Extensive Form을 직접 분석해야 한다.

§ 06

Nash Equilibrium in Extensive Form

Definition — NE in Extensive Form

Extensive Form 게임의 Nash Equilibrium은 해당 게임의 정규형 표현의 Nash Equilibrium으로 정의된다.

문제점: 비신뢰적 위협(Non-credible Threats)

Example — NE가 합리적이지 않은 경우
Anne ↓ / Bob →cecfdedf
a6, 96, 97, 57, 5
b5, 48, 65, 48, 6

정규형 NE: (a, ce), (a, cf), (b, cf) — 세 개

⚠️ 비신뢰적 NE

Bob이 실제로 c-e 분기에 도달하면 f를 선택한다 (6 > 5). Bob이 e를 선택한다는 위협은 비신뢰적(non-credible)이다. (a, ce)와 (a, cf)는 이 역동적 측면을 놓친다.

Backward Induction과 일치하는 유일한 NE: (b, cf)

Putin-Trump 예시

Example — Credibility of Threats
Putin ↓ / Trump →FightNot Fight
Invade−2, −2+1, −1
Not Invade0, 00, 0

NE가 두 개: (Invade, Not Fight)와 (Not Invade, Fight)

💡 어느 NE가 신뢰적인가?

Putin이 Invade를 선택한 상황에서, Trump는 Fight(−2)보다 Not Fight(−1)를 선택한다. 따라서 Trump의 "Fight" 위협은 비신뢰적이다. 신뢰적인 NE는 (Invade, Not Fight) 하나뿐.

§ 07

Backward Induction & Sequential Rationality

Definition — Sequential Rationality (순차적 합리성)

플레이어의 전략이 sequential rationality를 만족한다 ⟺ 자신이 움직이는 모든 정보집합에서, 해당 시점부터의 기대보수를 극대화하는 행동을 선택한다.

핵심: 실제로 도달하지 않을 것으로 예상되는 정보집합에서도 최적 행동을 명시해야 한다.

Backward Induction (역진귀납법) 절차

  1. 게임 트리의 가장 끝(마지막 결정 노드)에서 시작한다.
  2. 마지막 단계 플레이어는 자신의 보수를 극대화하는 행동을 선택한다.
  3. 이전 단계 플레이어는 뒤 단계 플레이어가 위처럼 행동할 것을 알고 자신의 보수를 극대화한다.
  4. 초기 노드까지 이 과정을 반복한다.
🔑 Backward Induction = CKSR (공통지식 가정)

BI를 적용한다는 것은 순차적 합리성의 공통지식(Common Knowledge of Sequential Rationality)을 가정하는 것이다.

  • 마지막 단계 플레이어는 합리적으로 행동한다
  • 그 전 단계 플레이어는 마지막 단계가 합리적임을 알고 행동한다
  • 그 전전 단계 플레이어는... (무한 반복)

Centipede Game — BI의 역설

Example — Centipede Game
P1 Pass P2 Pass P1 Pass P2 Pass P1 Pass P2 Pass 128, 32
Take↓
Take₁: 2, 0.5 Take₂: 1, 4 Take₁: 8, 2 Take₂: 4, 16 Take₁: 32, 8 Take₂: 16, 64
⚠️ BI vs CKSR

BI에 따르면 P1이 첫 노드에서 즉시 Take해야 한다. 하지만 현실 실험에서는 Pass가 많이 관찰된다. 왜?

CKSR ≠ CKR (공통지식 합리성): P1이 Pass를 선택하면, P2는 "P1이 비합리적인가?" 의문을 가질 수 있고, 이후 추론 체계 전체가 흔들릴 수 있다. CKSR은 강한 가정이다.

§ 08

Backward Induction의 성질과 한계

Properties of Backward Induction
  1. BI는 항상 정규형의 Nash Equilibrium을 도출하지만, 모든 NE가 BI에서 나오지는 않는다.
  2. 완전 정보의 유한 게임은 항상 BI로 풀 수 있다.
  3. Generic 게임(모든 경로가 서로 다른 보수를 가지는 게임)에서 해는 유일(unique)하다.

BI를 적용할 수 없는 두 가지 경우

① 무한 수평선 게임

Repeated Games처럼 종료 시점이 없으면 "마지막 단계"가 없어 BI 시작점이 없다.

② 불완전 정보 게임

Non-singleton 정보집합이 있으면 어느 노드에 있는지 모르므로 단순 BI 적용 불가.

→ 이 경우들을 다루기 위해 SPNE (Subgame Perfect Nash Equilibrium)가 필요하다.

BI의 한계 예시

Example — BI와 SPNE의 차이

다음 게임에서 Player 1이 먼저 E(exit) 또는 X(enter)를 선택한다. X를 선택하면 Player 1과 2가 2×2 동시 게임을 한다.

Player 1 선택: E → (2, 6) 즉시 종료 X → 2×2 동시 게임 진행 2×2 동시 게임 (X 선택 후): L R T (0, 1) (3, 2) B (−1, 3) (1, 5) BI 적용: 2×2 subgame의 NE: (T, R) → 보수 (3, 2) Player 1: E → (2, 6), X+NE → (3, 2) Player 1은 X를 선택 (3 > 2) BI 결과: (ET, R) 즉 X를 선택 후 T, 그리고 Bob은 R
§ 09

Subgame & SPNE 정의

Definition — Subgame (부분게임)

Extensive Form 게임의 일부로, 다음 세 조건을 모두 만족해야 한다:

  1. Subgame의 초기 노드는 singleton 정보집합에 속해야 한다
  2. 어떤 노드가 subgame에 속하면, 그 후계 노드(successors)도 모두 subgame에 속한다
  3. 정보집합의 어떤 노드가 subgame에 속하면, 그 정보집합의 모든 노드가 subgame에 속한다

게임 전체도 자기 자신의 subgame이다. 게임 전체 이외의 subgame을 proper subgame이라 한다.

Definition — Subgame Perfect Nash Equilibrium (SPNE)

전략 프로파일이 SPNE이다 ⟺ 게임의 모든 subgame에서 Nash Equilibrium을 유도한다.

SPNE ⊆ NE (SPNE는 NE의 정련(refinement)) 유한 완전정보 게임에서: SPNE = Backward Induction 해 무한/불완전정보 게임에서도 적용 가능

Subgame 개수 세기: 세 가지 예시

Subgame 개수 카운팅
게임구조Subgame 수이유
Anne → Bob (c/d / e/f) 완전 정보, 2단계 3개 전체 게임 + Bob이 a 선택 후 + Bob이 b 선택 후
P1 → (T/B / E) + P2 동시 non-singleton IS 있음 2개 전체 게임 + E 이후 동시 게임 (T/B는 non-singleton이라 subgame 불가)
P1 → A/B/C/D → P2 동시 P2가 항상 non-singleton IS 1개 P2의 모든 노드가 같은 IS → proper subgame 없음, 전체 게임만
§ 10

SPNE 계산 예시 (Step-by-Step)

SPNE 찾기 — P1(E/X) + 2×2 동시 게임

Subgame이 2개인 게임. P1이 먼저 E(exit=2,6) 또는 X(enter)를 결정하고, X 선택 시 P1과 P2가 동시에 T/B, L/R을 선택한다.

Step 1: Subgame 2 (proper subgame) 분석

P1 ↓ / P2 →LR
T0, 13, 2
B−1, 31, 5

Subgame 2의 NE: (T, R) → 보수 (3, 2)

Step 2: Subgame 1 (전체 게임) 분석

P1이 X를 선택하면 Subgame 2에서 NE (T,R)이 플레이됨을 안다 → P1의 X 선택 시 보수 = 3

P1이 E를 선택하면 보수 = 2

∴ P1은 X 선택 (3 > 2)

SPNE 결과

✓ Unique SPNE

(ET, R) — P1은 X를 선택하고, P1은 T를, P2는 R을 선택한다.

전체 게임 NE: (XT, L), (XB, L), (ET, R) 세 개 중 SPNE는 (ET, R)만. (XT, L)과 (XB, L)은 subgame에서 NE를 유도하지 못해 제거됨.

§ 11

SPNE의 한계: Forward Induction

SPNE도 완전한 해결책이 아닐 수 있다. 여러 SPNE가 존재할 때, 모두 동등하게 설득력 있는 것은 아니다.

Battle of Partners with Outside Option

Example — Putin & Trump (BoS with outside option)

Putin이 먼저 Book(=outside option, 보수 2,2)을 선택하거나 Go out을 선택한다. Go out 선택 시 Trump와 BoS 게임을 한다.

Putin(Go) ↓ / Trump →Bach(B)Stravinsky(S)
Bach(B)3, 10, 0
Stravinsky(S)0, 01, 3

Subgame(BoS)의 NE: (B, B) 와 (S, S)

전체 게임 SPNE:

  • (GB, B) — Putin: Go out 후 Bach; Trump: Bach → Putin 보수 3 > 2(Book) → Go out 선택 ✓
  • (BS, S) — Putin: Book 후 Stravinsky; Trump: Stravinsky → Putin 보수 2(Book) ≥ 1(S,S) → Book 선택 ✓
⚠️ (BS, S)는 설득력이 없다

Trump가 BoS에서 플레이하게 되었다면, Putin이 Book(보수 2)을 포기하고 들어온 것이다.

"왜 Putin이 들어왔을까?" → Putin은 Book보다 더 좋은 결과를 기대하고 들어온 것 → Putin은 (B, B)의 3을 노리고 들어온 것 → Trump의 best response = B

→ (BS, S)는 Forward Induction에 위배된다.

Forward Induction (전진 귀납)

플레이어가 과거의 비평형 행동을 관찰할 때, 그 행동이 상대방에게 합리적이었다면 어떤 의도에서였는지를 추론한다. 즉 과거 행동을 신호(signal)로 이용하여 미래 전략을 업데이트한다.

SPNE는 이러한 forward induction 추론을 포착하지 못하는 한계가 있다.

Example — Forward Induction 추론
게임: Putin이 A(보수 3,0) 또는 B를 선택. B 선택 시 Putin과 Trump가 동시 게임을 한다. E F G C (2,2) (2,1) (0,0) D (1,1) (1,2) (4,0) SPNE: (C, E)가 동시 게임의 유일 NE → (BC, E)가 유일 SPNE Forward Induction: Trump가 동시 게임에 진입했다면, Putin은 A(3,0)을 포기한 것. 응용: Stackelberg Competition

Setup — Stackelberg Duopoly
  • 역수요 함수: P(q₁, q₂) = a − (q₁ + q₂)
  • 한계비용: c > 0
  • Firm 1이 먼저 q₁을 선택한다
  • Firm 2가 q₁을 관찰한 후 q₂를 선택한다

→ Extensive Form 게임: Firm 1 → Firm 2 순차 결정

SPNE 계산: Backward Induction

【Step 1: Firm 2의 Best Response (임의의 q₁에 대해)】 π₂(q₂ | q₁) = [(a − (q₁ + q₂)) − c] · q₂ = [(a − c) − q₁ − q₂] · q₂ F.O.C.: (a − c) − q₁ − 2q₂ = 0 q₂^BR(q₁) = (a − c − q₁) / 2 ← 이것이 Firm 2의 전략 【Step 2: Firm 1의 최적화 (Firm 2의 BR을 알고)】 π₁(q₁) = [(a − c) − q₁ − q₂^BR(q₁)] · q₁ = [(a − c) − q₁ − (a−c−q₁)/2] · q₁ = [(a − c)/2 − q₁/2] · q₁ F.O.C.: (a − c)/2 − q₁ = 0 q₁* = (a − c) / 2 q₂* = q₂^BR(q₁*) = (a − c − (a−c)/2) / 2 = (a − c) / 4

🔑 First-mover Advantage
  • Firm 1(선도자)은 Cournot보다 더 많이 생산하고, 더 높은 이윤을 얻는다
  • Firm 2(추종자)는 Cournot보다 적게 생산하고, 낮은 이윤을 얻는다
  • 총생산량은 Cournot보다 증가 → 소비자 잉여 증가
  • 이는 Firm 1이 commitment(약속/공약) 효과를 활용하기 때문: q₁을 높게 고정하면 Firm 2는 반응함수에 따라 생산량을 줄일 수밖에 없다
